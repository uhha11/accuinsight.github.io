#Batch Pipeline

  1. 아래 내용에 대한 정확한 의미 확인 후 예시화면 추가할 것
  • File Path에 날짜 표현식을 입력하여 해당 날짜로 치환할 수 있음 : 불러오기 노드 공통
  1. HDFS불러오기 property에서 option 항목 입력방법 정리해서 매뉴얼에 반영 필요(Null 문자열처리, NaN 문자열 처리를 어떻게 하는지 잘 모르겠음)

  • ex) key: header, value: true = 헤더를 설정한다.
    key: nullValue, value: Null = Null 문자열 처리
    key: nanValue, value: NaN = NaN 문자열 처리

  • 텍스트 자체에 "NULL"이라는 문자열이 있을때 이를 실제 null로 볼것인지 설정하는 옵션이다. NaN=NULL 으로 하면 NULL 문자를 실제 na로 인식하게 변경한다. hive에서 null 데이터를 export할 경우 default로 NULL이란 문자가 들어가게 되므로 이를 실제 null값으로 변환하는 작업 -> 해당 예시를 만들어서 작업할 것

  1. pivot에서 6.overwriteSchema, 7.newSchema 이해하고 매뉴얼에 반영
  • 아래 문장 검토 후 매뉴얼 재반영(정확히 이해되지 않음) "6. overwriteSchema : 실행결과에 대한 스키마를 재정의할 경우 체크한다. 이전 스키마를 그대로 사용시 체크하지 않는다." "7. newSchema : 6번 Property를 체크하고 새로 생길 컬럼명을 입력한다. pivot 실행결과가 새롭게 생긴 컬럼에 선택된다."
  1. toJson 노드 실행결과 캡쳐해서 추가할 것

  1. customCode 내용 재정리

  2. dropsaturatecol 매뉴얼 재작성 및 캡쳐내용이 맞는 결과인지 확인 필요

  • 특정 컬럼에서 최소값이 1이라면 1이 특정 %만큼 존재하면 그 컬럼을 삭제한다. 예를 들어 ratio를 70%로 지정했을 때, 최소값으로 측정된 값이 1이고 이게 70%이상 분포하면 그 컬럼을 삭제한다.
  • dropsaturatecol node는 숫자 타입만 체크한다.

  1. nearzerovar 매뉴얼 재작성 : R의 nearzerovar와 동일한 기능이라고 표현하면 BP를 사용하는 고객사 분석가가 이해할 수 있는지...

[note]

  • 프로젝트 수정 화면에서 프로젝트에 할당한CPU/GPU 자원의 양을 확인할 수 있다.
  1. 아래는 인스턴스(GPU/CPU) 관련 추가설명으로 하나은행 매뉴얼에 추가했던 사항임. Public에서 필요한지 검토 필요

[note]

프로젝트 자원선택의 [GPU 개수]는 프로젝트에서 사용할 수 있는 전체 GPU 개수를, [인스턴스 자원]은 프로젝트에서 사용할 수 있는 전체 인스턴스 개수(GPU인스턴스+CPU인스턴스)를 의미한다. 예를 들어, GPU 개수를 2, 인스턴스 자원을 5로 입력할 경우 수행가능한 작업은 다음과 같다.

  • GPU 2개짜리 학습을 하나 돌릴 경우, cpu를 사용하는 인스턴스는 4개를 돌릴 수 있음
  • GPU 1개짜리 학습을 하나씩 2개 돌릴 경우, cpu를 사용하는 인스턴스는 3개를 돌릴 수 있음 프로젝트 생성 창의 GPU 개수는 프로젝트에서 사용할 수 있는 GPU의 최대개수를 지정(limit 설정)하는 의미이고, CPU 인스턴스의 경우 인스턴스당 한개의 CPU를 상용할 수 있다.
