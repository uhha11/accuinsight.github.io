글머리 기호 & 들여쓰기#

방안 1#
  1. 번호체계는 원문자를 사용하지 않고 아라비아 숫자 사용하면 좋을 것 같습니다.
  2. enter 입력시 자동 Numbering됩니다.
    • 그 이후 기호는 bullet list 입니다.
    • 번호체계는 하이픈(-) or asterisk(*) or + 입력시 점으로 표현됩니다.
    • shift 안 눌러도 되는 하이픈이 좋을 것 같습니다.
    • 하이픈도 enter 치면 자동으로 생성되네요.
      • enter 입력 후 탭 누르면 하위 카테고리를 만들 수 있습니다.
      • 자동으로 생성된 글머리 기호에서 다시 탭 입력하여 하위 카테고리를 만들 수 있습니다.
    • 글머리 기호에서 Backspace 입력하면 상위 카테고리로 이동합니다.
  3. 최상단 아라비아숫자는 안생기네요. 직접 다음 숫자를 입력해야 합니다.
  4. 숫자로 가지 않는 경우는 하이픈으로만 표기
  5. 문장과 문장사이에 엔터가 없었으면 좋겠다.
방안 2#
  1. 아라비아 숫자로 시작
    1. enter 후 자동넘버링 되는 숫자 앞에서 탭입력시 변경
    2. 하위 카테고리 번호에서 다시 enter 치면 자동 넘버링
      1. 자동 넘버링 번호 앞에서 탭 입력하면 새로운 번호 생성

[Flow 구성] 매뉴얼 포맷#

아래와 같은 포맷으로 작성하고 필요시 협의하여 변경

  1. 매뉴얼 구성 : Definition, Set, Example, Troubleshooing (헤더 size : H3(###))
  • Definition : 노드 정의 또는 특징
  • Set : setting, scheduler, parameter, property 설정항목.
    • 이중 setting, scheduler, parameter 는 아래 문장으로 대체
    [setting], [scheduler], [parameter] 설정은 [워크플로우 생성] > [설정]을 참고한다.
    • property 작성(헤더 size : H4(####))
  • Example : 사용예시. Example이 여러개일 경우 Example 1, Example 2 등으로 표시
  • Troubleshooting : 노드 설정/워크플로우 실행시 발생한 문제해결과정 기록
  1. 그 외 추가해야 할 설명들은 아래와 같이 작성
  • 부연설명, 용어정의, 출처 등 : [Note] : 제목 으로 설정
    • Blockquotes('>') 사용
    • Note는 어느 위치에서나 자유롭게 사용가능

      [Note] : 서울특별시 대기오염 측정정보 데이터
      상세설명 추가

  • source code : 노드에서 사용하는 r/python/pyspark/java/query/shell script 기재
    • Code Blocks(''') 사용
      CREATE EXTERNAL TABLE default.air_pollution(
      measurement_day STRING, -- 측정일시
  1. 참고
