#워크스페이스 관리

워크스페이스는 Rstudio, Jupyter Lab/Notebook이 설치된 가상 컴퓨팅 환경입니다.

  1. 워크스페이스 목록

    필드설명
    타입분석 Tool 타입 (Jupyter Lab/Notebook, R studio)
    이름워크스페이스를 대표하는 제목
    설명워크스페이스에 대한 설명
    상태현재 워크스페이스 상태 표기
    PENDING: 인스턴스 실행 및 리소스 설치가 실패된 상태
    RUNNING: 인스턴스가 정상적으로 실행 중인 상태
    TEMINATE: 리소스는 사용 중이지만 인스턴스가 종료 됨
    WAITING: 인스턴스를 초기화하는 상태
    STOPPED: 인스턴스 리소스는 삭제되고 메타정보만 저장된 상태
    N/A: 상태 알 수 없음
    CPU/GPU워크스페이스 별 할당 CPU 혹은 GPU core 수 및 실제 사용율(%) 표시
    Memory워크스페이스 별 할당 Memory 정보 및 실제 사용율(%) 표시
    생성일생성일, 시 표기
    Owner생성자 정보 표기
    재시작재시작 횟수
    기간생성 이후의 시간
    EndpointR studio/Jupyter 접속 URL 복사 / 바로가기
    Action시작 : 중지 상태의 워크스페이스의 인스턴스 리소스를 재생성
    중지: 인스턴스 리소스를 삭제 (인스턴스 사본 "저장 후 중지"/"저장하지 않고 중지" 선택 가능)
    수정
    삭제: 워크스페이스 생성/실행 관련 정보 모두 삭제. 동일 정보로 재생성/수정 불가
  1. 워크스페이스 생성
    • 분석 모델: R/Python 중 선택
    • Resource Quota 항목은 생성 후 수정 불가
    • 컨테이너 이미지 
      • R : rocker/rstudio (https://hub.docker.com/r/rocker/rstudio) 
      • Python : jupyter/base-notebook (https://hub.docker.com/r/jupyter/base-notebook)

#워크스페이스 상세 정보 관리

워크스페이스 상세페이지는 해당 인스턴스의 상세 정보를 조회하고 분석툴에서 생성한 파일을 조회하고 저장소로 저장할 수 있습니다.

  1. 요약

    • 워크스페이스 생성시 설정된 정보와 현재 인스턴스 상태, 발생한 이벤트 조회
    • Endpoint URL 바로가기 (분석툴 새창 open)지원, Jupyter/R 환경 생성 시 설정한 비밀번호 숨김/확인 기능 지원

    1) Jupyter Lab 바로가기 예시

  1. Experiment

    • Jupyter Lab/Notebook 에서 실행된 (AccuInsight SDK 활용) experiment 관리

      1) experiment 별 결과 확인

      2) Comparison : 2 ~ 10건 이내 experiment 에 대한 결과 비교 확인

      3) Edit Columns : UI를 통한 Parameter 수정

      4) 배포저장소 이동 : 최적의 experiment 정보 선택하여 배포저장소 (git repository)로 저장

  2. 작업폴더

    • 분석툴 홈 디렉토리의 파일 목록을 조회하고 파일을 선택하여 소스/배포저장소로 저장 가능
    • Rstudio : home/rstudio 
    • Jupyter Lab/Notebook : /home/work
  1. 계정폴더
    • 계정에 종속된 디렉토리의 파일 목록으로 R/Python 모델러 서비스내 모든 워크스페이스에서 공유되는 파일 목록

#워크스페이스 > 소스 저장소 저장

워크스페이스 작업폴더의 소스를 소스 저장소로 저장할 수 있습니다.

  1. 소스 저장소로 저장
    • 소스저장소는 분석 작업중인 소스코드 형상관리를 위한 저장소이며 워크스페이스 리소스가 삭제되더라도 소스저장소에 저장된 소스코드는 유지됨
    • 소스크기에 따라 저장하는 과정에서 다소 시간이 소요될 수 있음
  1. 신규소스 추가
    • 소스를 새로운 이름으로 소스 저장소에 저장
  1. 저장된 소스 업데이트
    • 기존 저장된 소스에 새로운 버전을 발행하여 소스 저장소에 저장
    • 버전은 시스템에서 자동으로 발행됨

#워크스페이스 > 배포 저장소 저장

워크스페이스 작업폴더의 모델을 배포 저장소로 저장할 수 있습니다.

  1. 배포저장소로 저장
    • ‘신규모델 추가’ 또는 ‘저장된 모델 업데이트’를 선택하여 저장
  1. 신규모델 추가
    • 모델을 새로운 이름으로 배포 저장소에 저장
  1. 저장된 모델 업데이트
    • 기존에 저장된 모델에 새로운 버전을 발행하여 배포 저장소에 저장
    • 배포 저장소로 저장시, 사용자는 모델에 명시적으로 버전을 지정해야 함

#Note

저장된 모델을 업데이트 시, 모델 버전은 이전보다 높은 버전이어야 함 (ex. 기존 버전: 1.0.1 인 경우 -> 신규 버전: 1.0.0 (x), 1.0.1(x), 1.1(o), 1.0.2(o))
