JupyterLab 확장

Git#

Git을 사용하여, 버전 제어
참조 - https://github.com/jupyterlab/jupyterlab-git

  1. 주요 내용
  • disableBranchWithChanges : 커밋되지 않은 변경 내용을 덮어 쓰거나 손실하지 않도록 보호함.
  • historyCount : 가장 최근부터 시작하여 기록 로그에 표시되는 커밋 수 표시.
  • refreshInterval : 변경을 위해 파일 시스템 폴링 사이의 시간 (밀리 초).
  • simpleStaging : 간단한 스테이징 개념을 가능하게함. 변경 사항을 시각적으로 간단하게 표현됨.

1) Git clone 방법

  • gitlab, github 등 기타 repository 관련 프로그램 clone 가능
  • 자체 branch 및 repository 관리가 가능하며, push는 불가능함.

Debugger#

Jupyter 소스 파일을 위한 시각적 debugger
참조 - https://github.com/jupyterlab/debugger

  1. 주요 내용
  • python 구문 사용 시, 단계별 디버그 확인가능
  • Kernel은 xpython만 가능
  • xpython은 여러개의 라이브러리가 포함된 python 인터프리터임.(참고 - http://xpython.org/)

1) Kernel 변경 방법

  • 최초 진입 시, Launcher에서 xpython 선택 가능

LSP#

IDE의 다양한 유사 기능 제공(코드 탐색, 호버 제안, 린터, 진단, 커널 없는 자동 완성 등..)
참조 - https://github.com/krassowski/jupyterlab-lsp

  1. 주요 내용
  • Language server 연동으로 폐쇄망에서 사용불가
  • 하단에 Fully initionalized 표시 되어야 정상 사용 가능
  1. 상세 기능 소개

  • Highlight References - 변수, 함수 등에 커서를 놓으면 모든 사용법이 강조 표시됨

  • Hover - 코드 조각 위로 마우스를 가져감. 밑줄이 나타나면 Ctrl 선택 기능 / 클래스 서명, 모듈 설명서 또는 언어 서버가 제공하는 기타 정보가 포함 된 툴팁을 얻을 수 있음.

  • Diagnostics - 심각한 오류에는 빨간색 밑줄이 있고 경고는 주황색 등이 있음. 밑줄이 표시된 코드 위로 마우스를 가져 가면 자세한 메시지가 표시됨.

  • Jump to Definition - 상황에 맞는 메뉴 항목을 사용하여 정의로 이동

  • Automatic Completion - 특정 문자 (예 : '.' 파이썬에서 (도트), 자동 완성을 트리거함

  • Automatic Signature Suggestions - 기능 서명이 자동으로 표시됨

  • Kernel-less Autocompletion - 커널 실행 없이 고급 정적 분석 자동 완성

  • Rename - 노트북과 파일 편집기에서 변수, 함수 등의 이름을 바꿈. 상황에 맞는 메뉴 옵션 또는 F2바로 가기를 사용하여 호출.

  • Diagnostics panel - 진단 패널을 사용하여 진단을 정렬하고 이동. JupyterLab 명령 팔레트 또는 컨텍스트 메뉴에서 "진단 패널 표시"를 검색.
