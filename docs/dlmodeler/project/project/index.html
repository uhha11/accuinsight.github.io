프로젝트#

About#

지금 시작해 보세요 버튼을 클릭하면 로그인 화면이 나타난다. 로그인 되어있는 경우엔 프로젝트 화면으로 이동한다. 로그인은 포탈 로그인과 연동된다.

상단 좌측, DL modeler 문구 옆의 역삼각형 버튼으로 AccuInsight+의 다른 서비스로 이동할 수 있다. 로그인 시 아래와 같이 메인 메뉴 탭이 나타나며, Select Project 박스에는 현재 사용 중인 프로젝트가 나타난다.

Admin 메뉴는 Admin 계정에만 나타나며, 메뉴에서 사용자를 관리할 수 있다. 프로젝트 생성 후, 프로젝트를 선택해야 분석 메뉴(데이터, 작업, 배포)가 활성화된다.

프로젝트#

프로젝트는 사용자가 DL Modeler를 사용하기 위한 최상위 추상 그룹의 개념으로 데이터 관리, 학습, 배포 등 모든 작업이 수행되는 공간이다. 사용자는 여러개의 프로젝트를 생성 할 수 있으며, 기 생성한 각 프로젝트 안에서 DL Modeler의 데이터, 작업, 배포 기능을 사용할 수 있다.

  • 상단 메뉴 바에서 Select Project를 클릭하면, 현재 사용 중인 프로젝트를 조회 및 선택할 수 있다.
  • 프로젝트 관리 버튼 클릭 시 프로젝트 관리 페이지로 진입한다.

프로젝트 관리#

프로젝트 관리 화면에서 분석 주제 별 프로젝트 리스트를 조회 할 수 있다. 등록된 프로젝트가 많을 경우 우측 검색기능에서 프로젝트 명을 직접 입력해서 조회할 수 있다.

프로젝트 카드 우상단 메뉴 박스에서 기 생성한 프로젝트의 편집, 삭제가 가능하다.

  • 프로젝트 삭제 클릭 시, 해당 프로젝트에 속한 데이터, 작업, 배포 정보가 모두 삭제된다.
  • 각 프로젝트 카드에서, 보유한 데이터, 생성한 작업, 학습, 배포 모델의 건 수를 조회할 수 있으며, 생성한 유저정보와 생성시간을 파악 할 수 있다.

프로젝트 생성화면에 있는 [프로젝트 생성]버튼을 눌러 프로젝트를 생성한다.

  • 프로젝트 생성 시 프로젝트를 같이 사용할 사용자를 선택하여 추가 할 수 있다
  • 이미 존재하는 프로젝트 명은 사용할 수 없고 영문(소문자), -(하이폰), 숫자만 사용할 수 있으며 하이폰과 숫자는 프로젝트 명의 처음에 올 수 없다.

프로젝트 수정#

프로젝트 카드에서 프로젝트 편집을 클릭하면 프로젝트 편집 창이 나타난다. 프로젝트에 접근 가능한 사용자, 프로젝트 관리자를 변경할 수 있다.

특정 프로젝트를 선택하면 작업, 배포, 데이터 메뉴가 활성화 된다. 프로젝트에 귀속된 작업, 배포, 데이터를 관리한다.

Note On-Premise 환경의 경우 가용자원이 유한하므로 프로젝트 생성시 인스턴스를 선택한다.
프로젝트 생성/수정시 GPU/CPU 인스턴스 및 개수를 선택할 수 있다.

  1. 프로젝트 생성시
  • 프로젝트 자원선택의 [GPU 개수]는 프로젝트에서 사용할 수 있는 전체 GPU 개수를, [인스턴스 자원]은 프로젝트에서 사용할 수 있는 전체 인스턴스 개수(GPU인스턴스+CPU인스턴스)를 의미한다.
  • 예를 들어, GPU 개수를 2, 인스턴스 자원을 5로 입력할 경우 수행가능한 작업은 다음과 같다.
    • GPU 2개짜리 학습을 하나 돌릴 경우, cpu를 사용하는 인스턴스는 4개를 돌릴 수 있음
    • GPU 1개짜리 학습을 하나씩 2개 돌릴 경우, cpu를 사용하는 인스턴스는 3개를 돌릴 수 있음
  • 프로젝트 생성 창의 GPU 개수는 프로젝트에서 사용할 수 있는 GPU의 최대개수를 지정(limit 설정)하는 의미이고, CPU 인스턴스의 경우 인스턴스당 한개의 CPU를 사용할 수 있다.
  1. 프로젝트 수정시
  • 프로젝트 수정 화면에서 프로젝트에 할당한CPU/GPU 자원의 양을 확인할 수 있다.
