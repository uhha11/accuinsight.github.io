Job

작업#

작업 기능은 DL Modeler의 컴퓨팅 인스턴스 자원을 활용하여, 데이터의 분석 및 모델을 활용할 수 있는 기능이다. 작업 방식에 따라, 특정 기능에 종속된 Basic 과 사용자가 직접 Jupyter Notebook을 할당 받는 Custom 기능이 있다.

작업관리#

사용자가 생성한 작업에 대한 정보를 조회할 수 있다.

  • 프로젝트 > 작업을 클릭하면 작업 관리 화면을 조회할 수 있다..
  • 작업생성 버튼을 클릭하면 프로젝트에 종속 되는 신규 작업을 생성한다.
  • 작업 생성 방식은 Basic과 Custom 두 가지 방식을 선택할 수 있다.
  • Action > 재생/정지 버튼으로 Custom 방식으로 생성하는 instance를 점유 및 해제 가능하다.
  • 휴지통 버튼은 작업과 작업 하위의 모든 학습을 삭제한다.

작업 생성#

작업생성에서, 특정 기능에 종속된 Basic 과 사용자가 직접 Jupyter Notebook을 할당 받는 Custom 기능을 선택할 수있다.

  • Basic은 DL modeler에서 제공되는 알고리즘을 통해 UI와 parameter만 입력해서 딥러닝 학습하는 기능을 제공한다.
  • Custom은 jupyter notebook을 기반으로 작성한 코드를 로드해서 딥러닝 학습하는 기능을 제공한다.

Basic 작업 생성#

Basic 생성에서는 built-in 알고리즘을 선택할 수 있다.

  • 알고리즘 및 학습에 사용할 데이터를 선택할 수 있으며 Basic알고리즘은 아래와 같다. 데이터 선택 란은 선택하는 알고리즘에 따라 변한다.
  • 작업과 작업 설명, 알고리즘, 데이터를 모두 선택하면 생성 버튼이 활성화된다.

Basic 알고리즘#

텍스트 분류기#

텍스트와 해당 텍스트의 레이블을 학습하여 분류하는 RNN 기반 알고리즘이다.

  • 텍스트 데이터와 레이블 데이터가 필요하다. 텍스트와 레이블 데이터는 파일 이름이 동일해야 한다. 또한, 여러 파일을 동시에 학습할 수 있다.
  • 텍스트 데이터는 단어 벡터화와 문자 벡터화 전처리를 해서 사용해야 하며, 레이블 데이터는 각 행이 특정 카테고리에 속할 확률 값으로 구성된다.
이미지 분류기#

이미지와 레이블을 학습하여 분류하는 CNN 기반 알고리즘이다.

  • 이미지 데이터와 레이블 데이터가 필요하다. 이미지는 레이블에 따라 폴더로 구분한 형태로 만들어서 압축 파일로 한번에 업로드한다.
  • 레이블은 알파벳 순으로 0부터 1씩 증가하는 숫자로 변환된다.
word-embedding#

단어를 벡터로 매핑시켜 주는 word2vec 알고리즘으로, 타겟 단어로 주변 단어를 예측하는 Skip-gram 방식을 기반으로 한다.

  • 변환된 임베딩 벡터 모음이 .npy 파일로 생성되며, 학습 배포를 지원하지 않는다.
  • input은 텍스트 데이터다. 별도의 전처리가 필요하지는 않지만, 전처리가 잘 되어있을수록 성능이 높아진다.
수치 예측 모델#

RNN LSTM 기반으로 시계열 데이터의 미래 값을 예측해 보는 알고리즘이다.

  • n개의 벡터(data_dim)로 이루어진 시계열 데이터 csv 파일이 필요하다.
이미지 오브젝트 검출#

클래스 파일에 주어진 오브젝트를 이미지에서 검출해주는 알고리즘이다.

  • 주석과 클래스의 csv 데이터 파일이 필요하다.
  • 주석 파일에는 전체 경로를 포함하는 이미지 파일명, 이미지에 나타난 오브젝트 좌표(x1, y1, x2, y2), 클래스 명이 들어가고 클래스 파일에는 클래스 명과 ID 값이 들어간다.

Custom 작업 생성#

Custom 작업에서는 사용자가 직접 코딩할 수 있는 파이썬 기반 Jupyter Notebook 인터프리터를 제공하며, GPU를 할당 받아 Jupyter 인스턴스를 기동 할 수 있다.

  • 작업명과 설명을 작성하고, jupyter notebook작업을 기동 할 GPU / CPU instance를 선택한다. custom 작업을 생성하면 GPU 또는 CPU 자원을 하나씩 점유하며, 프로젝트에 할당 받은 자원만큼만 생성할 수 있다.
  • 생성 버튼을 클릭하여 작업을 생성한다.

작업 상세#

기 생성 된 작업을 클릭하면, 작업 상세 화면이 나타난다. 작업 상세 화면에서는 작업에 대한 학습 관리/ 상세 조회 및 Custom기능의 경우 Jupyter Notebook 에 접속할 수 있는 엔드포인트를 제공한다.

  • 작업의 자원 할당을 active / deactive / standby 로 나타내며 각각 사용가능함, 사용불가, 자원이 없어 할당 대기중 임을 의미한다.
  • 학습이 진행 중인 경우, 학습 상태 아이콘에 마우스를 올려놓으면 학습 진행률을 확인할 수 있다.
  • 학습 성능지표를 수치로 제공하여 학습 결과를 실시간으로 확인할 수 있다.
  • accuracy, precision, recall은 숫자가 높은 방향으로, mse, mae, loss는 낮은 방향으로 갈수록 학습이 잘 진행되고 있음을 나타낸다.
  • 생성 된 학습에 대해 정지 버튼과 휴지통 버튼으로 학습을 정지 및 삭제가 가능하다.
  • 완료 된 학습에 대해 배포 버튼을 통해 배포가 가능하다.
  • 학습 명과 상태 및 기간을 입력하여 검색이 가능하다.
  • 작업 수정으로 작업설명을 수정 할 수 있으며 작업 삭제 버튼으로 작업삭제가 가능하다. (학습들도 함께 삭제된다)

  • 상세 탭에서는 작업의 상세 내용 확인 및 작업 내용 수정 기능을 제공한다.
  • 베이직 알고리즘의 경우, 어떤 데이터를 선택하여 작업을 생성했는 지도 보여준다.

학습 배포#

학습 배포 화면은 학습 관리 화면의 기 생성된 학습 중 배포 버튼 클릭 시 나타난다. 학습 배포 시 생성된 모델은 최종모델, 또는 Checkpoint 별 배포를 선택 할 수 있다.

  • 배포 버튼을 누르면 배포 창이 활성화된다.
  • 최종 모델 또는 체크포인트로 배포가 가능하다. 체크포인트는 학습 1회(1 epochs) 마다 생기는 중간 모델 파일이다.
  • 배포서버명과 배포서버 설명을 입력한다.
  • 배포 인스턴스(GPU/CPU)를 선택한다.
  • 배포에 활용할 최소 인스턴스 개수를 선택한다. 여기에서 선택한 인스턴스 개수 만큼 배포 서버가 생성된다.
  • 인스턴스 자동 확장(auto-scaling) 기능 사용 여부를 선택한다. 자동 확장 기능을 사용하면, 배포 API에 일정 기준 이상(현재는 60%)의 CPU부하가 발생할 경우 자동으로 배포 서버 개수를 늘리고, 부하가 줄어들면 다시 원래대로(최소 개수)로 돌아간다. 사용을 선택할 경우, 최대 몇 개까지 확장할 지, 숫자를 입력해 준다.
  • 모델(또는 체크포인트) 파일 경로검색을 클릭해 학습 후 생성 된 모델 파일을 선택한다.
  • 베이직 알고리즘의 경우, 전처리 API가 연결된다. 데이터 화면에서 처리한 전처리 모듈이 연결된다.
  • 배포 버튼을 클릭하면 모델이 호스팅되어 서비스 상태가 된다.
  • 배포의 수정과 삭제는 배포 관리 페이지에서 가능하다. (2.5. 배포 참조)

  • 예 버튼을 클릭하면 배포 관리 페이지로 이동된다.

Custom notebook#

Custom 작업은 작업생성 > Custom 을 선택 시 생성 할 수 있다. Jupyter Notebook을 통해 사용자가 인터프리터 형식 개발을 진행 할 수 있다. 생성한 Custom 작업은 ‘Jupyter Notebook’ 버튼을 통해 아래와 같이 notebook에 진입할 수 있다.

  • Jupyter Notebook token및 이동 팝업 예시 화면이다.

  • custom 작업의 경우, jupyter notebook을 클릭하면 jupyter notebook이 실행된다. (작업이 active상태일 때 활성화 됨)

  • Notebook 선택 가능한 Kernel은 Python3 이다.

  • 단, 작업 UI에서 멀티 GPU로 학습하려면 3장에서 서술되는 programming guide를 참고하여 작성한다. 지원 프레임워크는 총 3개(Tensorflow, Keras, Pytorch)로, 각 Model function을 정해진 양식에 맞게 구현해야 한다. 또는, 직접 멀티 GPU를 사용하는 horovod 코드를 작성해야 한다.

  • custom 학습을 위한 샘플 코드를 sample 폴더에서 볼 수 있다.

  • python 패키지를 추가로 활용할 경우, notebook 상에서 다음 명령어를 활용할 수 있다.

    • !pip install {패키지 명}
    • !pip install {설치 파일 명} (패키지 파일 jupyter notebook 업로드 필요)

  • 단, 새 패키지는 jupyter notebook 상에서만 적용되고, DL Modeler를 이용한 학습과 배포에 패키지를 활용해야 할 경우 운영자에게 패키지 추가를 별도로 요청해야 한다. (jupyter notebook과 학습, 배포는 별개로 운영되기 때문)

학습생성#

생성한 Basic/Custom작업을 클릭하고, 상세화면 진입 한 뒤 학습 생성 버튼을 클릭하면 학습생성 팝업이 나타난다

  • 학습 추가버튼을 통해 다양한 parameter set을 동시에 학습 가능하다.
  • 각 학습의 이름을 입력한 후 생성 버튼을 누르면 지정한 parameter set 수 만큼 학습되며, 작업상세 목록에 학습 리스트가 나타난다.
  • 학습 생성에 활용되는 총 인스턴스 개수가 프로젝트에 할당된 자원을 초과할 경우, 학습 생성이 불가능하다. (ex. gpu 2개로 학습을 2개 생성할 경우, 프로젝트에 gpu가 4개 이상 할당되어 있어야 함)
  • 학습은 학습대기 -> 학습 중 -> 학습완료로 진행된다

학습 관리> 학습생성 > Basic#

  • 학습 인스턴스에서 학습 할 gpu수 혹은 cpu사용을 선택하고, hyper-parameter를 입력한다. 처음 보여지는 숫자는 기본으로 세팅된 hyper-parameter 값이다.
  • DL Modeler에서는 멀티 gpu 프레임워크로 horovod를 이용한다. gpu를 선택할 경우, 내장되어 있는 horovod 코드가 학습이 진행되면서 자동으로 추가된다. 따라서 사용자는 활용할 gpu 개수만 입력하면 된다.
  • gpu를 2개 이상 사용할 경우, 동일한 모양의 학습 팟을 띄워야 하는 horovod특성 때문에 짝수 개로만 사용이 가능하다. (ex. 2개, 4개, 6개…)
  • 학습 데이터는 작업 생성 시 입력했던 데이터가 된다. 데이터를 변경해야 할 경우, 작업을 새로 생성해야 한다.

학습 관리> 학습생성 > Custom#

  • jupyter notebook에서 3장의 프로그래밍 가이드에 따라 적절히 작성 된 코드를 경로검색 버튼을 클릭하여 불러올 수 있다.
  • 코드를 불러오기 전에 코드 작성 시 사용한 딥러닝 프레임워크를 먼저 선택해야 한다. 선택 가능한 프레임워크는 총 4가지(Tensorflow, Keras, Pytorch, Horovod)이다.
  • DL Modeler에서는 멀티 gpu 프레임워크로 horovod를 이용한다. 딥러닝 프레임워크(Tensorflow, Keras, Pytorch)를 사용할 경우, 학습 인스턴스로 gpu를 선택하면 내장되어 있는 horovod 코드가 학습이 진행되면서 자동으로 추가된다. 따라서 사용자는 활용할 gpu 개수만 입력하면 된다.
  • Horovod를 선택하면, Horovod코드를 직접 작성하여 멀티 gpu를 이용할 수 있다. (DL Modeler 내부 코드를 활용하지 않음)
  • gpu를 2개 이상 사용할 경우, 동일한 모양의 학습 팟을 띄워야 하는 horovod특성 때문에 짝수 개로만 사용이 가능하다. (ex. 2개, 4개, 6개…)
  • 코드에서 params를 dict형태로 작성하면 불러올 때 작성한 hyper-parameter가 UI에 노출된다.
  • ipynb 파일을 저장한 후 파일을 불러와야 정상적으로 hyper-parameter가 출력된다.

  • Horovod 코드를 직접 작성하는 경우, UI 상에서의 hyper-parameter 설정을 지원하지 않으므로 코드 내부에서 hyper-parameter를 설정해야 한다.

학습 상세#

작업 관리 > 학습 관리 화면에서 해당 학습을 클릭하면 현재 학습한 학습상세 화면을 조회 할 수 있다. 본 팝업에서는 학습에 따른 정확도 결과 그래프와 학습에 사용된 데이터의 상세정보를 확인 할 수 있다.

  • 작업상세 화면에서 학습 리스트의 특정 학습을 클릭하면 학습 상세화면이 나타난다.
  • 결과 그래프 탭에서 학습 진행률과 각 epoch 당 loss graph를 확인 할 수 있다. Horovod를 선택한 경우 학습 진행 정보 그래프를 지원하지 않는다.
  • 학습 중 상태의 경우, 학습 중지 버튼과 그래프 리프레쉬 버튼이 활성화된다. 학습 중지 버튼으로 진행 중인 학습을 멈출 수 있다. 학습이 중지되면 학습 중지 버튼은 학습 점유(학습 시작) 버튼으로 바뀐다.

  • 학습 진행이 완료(또는 실패)되면, 진행 상태 자세히 보기 버튼을 클릭하여 학습에 사용한 python 코드의 로그를 볼 수 있다. 학습 실패가 발생한 경우, 로그를 확인하여 학습 코드를 수정하는 등 보완 작업을 진행한다.

  • 완료 된 학습은 배포 버튼으로 배포할 수 있다.
  • 배포 버튼은 작업 상세 화면의 배포 버튼과 동일하게 작동한다.

  • 상세 정보 탭을 클릭하면 학습 상세 정보 화면이 나타난다.
  • 학습 정보 조회, 하이퍼 파라미터 조회 및 재학습 기능을 제공한다
  • 재학습은 학습 완료된 모델을 추가로 학습하는 기능으로, “파라미터를 유지한 상태로 재학습”과 “파라미터를 유지하지 않은 상태로 재학습” 두 가지 방식이 있다.
