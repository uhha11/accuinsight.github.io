계정 관리

계정 관리#

  1. Account
    • 계정의 ID 입니다. 자동 생성되는 시퀀스 번호
  2. 클라우드
    • 계정의 클라우드 정보
  3. 설명
    • 계정 등록 시 등록한 설명
  4. 등록일
    • 계정을 등록한 시간
  5. 권한
    • 권한 부여 기능이 있음
  6. Action
    • 수정, 삭제 기능이 있음

Note

계정관리 AWS 의 IAM 권한을 통해 Role 방식으로 권한이 주어져 있어 계정등록은 따로 하지 않아도 됩니다.

계정 등록#

  1. 검색 필터

    • 클라우드, 등록일로 리스트를 검색할 수 있음

  2. 계정 등록

    • 계정 등록 팝업을 통해 계정을 등록
    • GCP 계정 등록 시 JSON으로 작성된 KeyFile을 등록 해야 하고 요금 정보가 Export되는 Dataset의 Table을 선택해야 함, Billing Dataset을 입력하지 않으면 빌링 정보가 수집되지 않음
      • 사전에 GCP Master 계정에서 BigQuery로 Export 설정을 해야 Table 정보가 생성 (참고 : https://cloud.google.com/billing/docs/how-to/export-data-bigquery)
      • "테스트" 버튼을 통해 테스트가 완료 되어야 등록이 가능, 권한이 없을 경우 PERMISSION_DENIED 과 같은 오류 팝업이 발생 할 수 있음
      • "테스트" 버튼은 입력 내용에 대한 단순 검증으로 오류 팝업이 발생 하지 않아도 충분한 권한이 없으면 클러스터를 생성할 수 없음
    • [AWS 필요 권한]
      • EMR 생성/수정/삭제, VPC 조회, Keypair 조회, S3 업로드/다운로드, 요금 조회
    • [GCP 필요 권한]
      • Dataproc 생성/수정/삭제, Network 조회, Storage 업로드/다운로드, Dataset 조회, BigQuery 조회

Note

계정 수정, 삭제 버튼 Role 방식의 계정관리는 계정의 수정, 삭제를 할 수 없습니다.

계정 삭제/리플레쉬/권한#

  1. 삭제
    • 체크 박스가 체크된 계정을 삭제
  2. 리플레쉬
    • 계정 리스트를 갱신
  3. 권한
    • 권한을 부여
    • 계정의 관리자만 조회/생성 권한을 부여 할 수 있음 권한이 부여 된 사용자는 해당 계정으로 클러스터를 조회/생성 할 수 있음
    • [권한]
      • 없음 : 아무 권한 없음
      • 조회 : 조회만 가능
      • 조회/생성 : 조회와 생성/삭제가 가능
  4. Action
    • 수정 : 계정을 수정
    • 삭제 : 계정을 삭제
