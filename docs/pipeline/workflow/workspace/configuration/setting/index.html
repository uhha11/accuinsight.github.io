Setting

#기본설정

워크플로우 생성 화면에서 우측 패널(setting, scheduler, parameter, property) 개요 및 입력방법에 대해 설명한다. 사용자가 생성하는 Workflow에 공통적으로 설정되는 값으로, 입력방법을 잘 모를 경우 ⓘ 버튼에 마우를 올려놓으면 메뉴에 대한 설명정보를 제공한다.

#Setting

최초에 화면에 진입하거나 setting 패널을 클릭하면 하단 패널이 확장된다.

Setting은 현재 워크플로우의 속성을 입력할 수 있는 폼 형태로 구성되어있다. 처음에는 필수입력항목(붉은색 *마크가 붙어있다.)만 노출되고, 더보기 버튼을 클릭 했을 경우 나머지 모든 항목들을 보여준다.

  1. cluster : 워크플로우 실행 클러스터를 선택한다. 최초 작성시 선택하고, 관리메뉴에서 워크플로우 실행시 변경 가능하다.

  2. notifyLevel : 사용자에게 고지되는 실행 알림의 level을 정할 수 있다. Info로 선택할 경우에 정상 종료 되어도 실행 알림을 받을 수 있다.

  3. icosLogging : enable 할 경우 실행 로그를 사용자 Object Storage에 저장가능하다.

  4. workflow : libPath는 작업 시에 참조하는 library의 hdfs 상의 경로를 의미한다

    • name: workflow name을 입력한다(designer 상단의 “이름” 기능과 동일).
    • libPath : 워크플로우 수행을 위해 필요한 별도 라이브러리(JAR 등)가 있을 경우 해당 libPath 경로를 지정할 수 있다(default : /tmp/lib/batch).
    • useSystemLibPath : Batch Pipeline에서 제공하는 시스템 라이브러리를 사용하겠다는 의미다(default : 체크상태). 시스템 lib를 사용하지 않을 경우 useSystemLibPath를 비활성화 할 수 있다.

Note

워크플로우 실행을 위해 사용자가 라이브러리를 직접 구현했거나 JDBC 액션을 위한 별도 드라이버, 커넥터 등을 사용해야 할 경우 libpath 경로에 필요한 파일을 등록한다.
