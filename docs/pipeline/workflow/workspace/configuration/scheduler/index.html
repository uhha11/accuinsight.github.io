Scheduler#

워크플로우 스케줄링 설정을 위해 필요한 기능들을 제공합니다. 워크플로우 생성화면으로 이동 후 Scheduler 탭을 선택합니다. Scheduler 패널의 [더보기+] 버튼을 누르면 입력가능한 전체 항목을 볼 수 있습니다.

  1. enable : enable 항목 체크시 scheduler 탭에서 설정한 사항에 맞게 워크플로우 job 실행

  2. timezone : timezone 설정(기본값 : kst)

  3. start : 시작 일시 설정(현재 시간보다 이후로 지정)

  4. end : 종료 일시 설정

  5. frequency : Time Interval 기반으로 워크플로우 job 실행. Basic, Cron 등 두 가지 InputType 제공

    • Basic : frequencyNumber에 실행주기(숫자값), frequencyUnit에 minutes, hours, days, months 중 선택 하여 주기 설정 가능. frequencyUnit을 minutes로 frequencyNumber를 5로 입력한 경우 5분 주기로 워크플로우 job을 실행하겠다는 의미. 워크플로우 실행 최소주기는 5분으로 그 이하로 설정할 수 없음
    • Cron : Unix Crontab 스타일로 스케줄 설정. 평일 10분 마다 워크플로우 실행시 [0/10 ? 1-5 *]과 같이 입력

  6. controls : timeout, concurrency, throttle, execution 설정

    • timeout : 타임아웃 시간을 설정(기본값 : -1)

      timeout설명
      -1timeout 미설정. 즉 워크플로우 작업이 가능할 때까지 무한정 대기
      0workflow가 정상적이지 않을 경우 즉시 종료

    • concurrency : 동시에 실행(running)시키는 작업 수

    • throttle : 특정 시점에 실행 대기(waiting) 중인 작업 수

    • execution : 인스턴스들의 실행 규칙을 설정

      execution설명
      FIFO순서대로 실행
      LIFO가장 나중에 들어온 것부터 실행
      ONLYLAST마지막 작업만 실행하고 나머지 작업은 실행하지 않음

  7. datasets : 미리 작성해둔 dataset을 불러와서 파일이 존재하거나 디렉토리 네이밍 규칙 등으로 조건에 따른 스케줄 설정

    • inputEvents

    • name: input-dataset 이름
    • uriTemplate: dataset이 위치한 HDFS 경로
    • frequency: 수행 주기 (minutes, hours, days, months 중 택1)
    • initialInstance: 첫번째 인스턴스 시점 (해당 시점 이전에 생성된 인스턴스는 무시)
    • timezone: timezone 정보
    • doneFlag: 필드를 통해서 코디네이터가 해당 데이터셋이 처리할 준비가 되었는지를 알려줌(_SUCCESS, DIRECTORY 중 택1)

    • outputEvents

    • name: output-dataset 이름
    • uriTemplate: dataset이 위치한 HDFS 경로
    • frequency: 수행 주기 (minutes, hours, days, months 중 택1)
    • initialInstance: 첫번째 인스턴스 시점 (해당 시점 이전에 생성된 인스턴스는 무시)
    • timezone: timezone 정보
    • doneFlag: 필드를 통해서 코디네이터가 해당 데이터셋이 처리할 준비가 되었는지를 알려줌(_SUCCESS, DIRECTORY 중 택1)
