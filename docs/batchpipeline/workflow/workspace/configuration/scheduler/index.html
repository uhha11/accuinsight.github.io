Scheduler

#Scheduler

Scheduler 패널을 클릭하면 하단 패널이 확장된다. Scheduler 패널의 [더보기+] 버튼을 누르면 입력가능한 전체 항목을 볼 수 있다.

  1. enable : enable 항목에 체크할 경우 scheduler 패널에서 설정한 사항에 맞게 워크플로우 job을 실행한다.

  2. timezone : timezone 설정할 수 있으며 기본값은 kst 이다.

  3. start : 시작 일시를 설정할 수 있다(시작 일시는 현재 일시 보다 이후로 지정해야 한다).

  4. end : 종료 일시를 설정할 수 있다.

  5. frequency : Time Interval 기반으로 워크플로우 job을 실행한다. Basic, Cron 등 두 가지 InputType을 제공한다.

    • InputType을 Basic으로 선택할 경우 frequencyNumber에 실행주기(숫자값), frequencyUnit에 minutes, hours, days, months 중 선택 하여 주기를 설정 할 수 있다. frequencyUnit을 minutes로 frequencyNumber를 5로 입력할 경우 5분 주기로 워크플로우 job을 실행하겠다는 의미이다. 워크플로우 실행의 최소주기는 5분으로 그 이하로 설정할 수 없다.
    • InputType이 Cron이면 Unix Crontab 스타일로 스케줄을 설정할 수 있다. 평일 10분 마다 워크플로우가 실행된다면 [0/10 ? 1-5 *]과 같이 입력한다.

  6. controls : timeout, concurrency, throttle, execution 을 설정할 수 있다.

    • timeout : 타임아웃 시간을 설정한다. default는 -1이다.
    timeout설명
    -1timeout 미설정. 즉 워크플로우 작업이 가능할 때까지 무한정 대기함
    0workflow가 정상적이지 않을 경우 즉시 종료함
    • concurrency : 동시에 실행(running)시키는 작업 수
    • throttle : 특정 시점에 실행 대기(waiting) 중인 작업 수
    • execution : 인스턴스들의 실행 규칙을 설정
    execution설명
    FIFO순서대로 실행
    LIFO가장 나중에 들어온 것부터 실행
    ONLYLAST마지막 작업만 실행하고 나머지 작업은 실행하지 않음

  7. datasets : 미리 작성해둔 dataset을 불러와서 파일이 존재하거나 디렉토리 네이밍 규칙 등으로 조건에 따른 스케줄을 설정할 수 있다.

    • inputEvents

    • name: input-dataset 이름
    • uriTemplate: dataset이 위치한 HDFS 경로
    • frequency: 수행 주기 (minutes, hours, days, months 중 택1)
    • initialInstance: 첫번째 인스턴스 시점 (해당 시점 이전에 생성된 인스턴스는 무시)
    • timezone: timezone 정보
    • doneFlag: 필드를 통해서 코디네이터가 해당 데이터셋이 처리할 준비가 되었는지를 알려줌(_SUCCESS, DIRECTORY 중 택1)

    • outputEvents

    • name: output-dataset 이름
    • uriTemplate: dataset이 위치한 HDFS 경로
    • frequency: 수행 주기 (minutes, hours, days, months 중 택1)
    • initialInstance: 첫번째 인스턴스 시점 (해당 시점 이전에 생성된 인스턴스는 무시)
    • timezone: timezone 정보
    • doneFlag: 필드를 통해서 코디네이터가 해당 데이터셋이 처리할 준비가 되었는지를 알려줌(_SUCCESS, DIRECTORY 중 택1)
