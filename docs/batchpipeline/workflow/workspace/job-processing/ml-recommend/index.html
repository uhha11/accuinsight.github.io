mlRecommend

Definition#

ML Modeler서비스에서 기 저장된 모델을 사용하여 추천 작업을 실행한다. [Flow구성]노드 중 [mlRecommend]노드를 drag & drop 한 후 Property 항목을 입력한다. Property 패널의 [더보기+] 버튼을 누르면 입력가능한 전체 Property 항목을 볼 수 있다.

Set#

[setting], [scheduler], [parameter] 설정은 [워크플로우 생성] > [설정]을 참고한다.

property#

[Node Description] 작성 중인 노드명 입력

  1. master : mlRecommend 실행할 master 선택 (yarn-cluster, yarn-client, local 중 선택)
  2. name : appName입력
  3. useTrainNodeModel : 연결된 mlTrain 노드의 학습결과 모델 사용 여부
  4. modelPath : 모델 경로 (팝업열기 : HDFS, S3 File browser 팝업에서 모델 경로를 선택, ml model 불러오기 : 기존에 등록된 model을 선택)
  5. modelPathcredential : modelPath가 s3인 경우 credential 정보 (modelPath 설정 시 자동셋팅됨)
  6. savePath: 저장할 경로 선택 (HDFS, S3 File browser을 제공-팝업에서 해당 경로를 선택)
  7. savePathcredential : savePath가 s3인 경우 credential 정보 (savePath 설정 시 자동셋팅됨)
  8. saveDataSetFormat : 저장할 데이터셋 포맷 선택 (parquet, csv, orc 중 선택)
  9. saveDataSetDelimiter : 저장할 데이터셋 구분자 입력
  10. saveDataSetHeaderExists : 저장할 데이터셋 header가 있는지 여부 (TRUE, FALSE 중 선택)
  11. recommendType : 추천 유형 선택 (recommendForAllUsers 사용자별 상품 추천, recommendForAllItems 상품별 사용자 추천 중 선택)
  12. numRecommends : 추천할 항목 수 입력
  13. userCol : user 컬럼 설정 (저장할 데이터의 사용자 id 저장 컬럼명 입력)
  14. itemCol : item 컬럼 설정 (저장할 데이터의 상품 id 저장 컬럼명 입력)
  15. predRatingCol : 저장할 데이터의 rating 예측 컬럼명 입력
  16. sparkOpts : spark 옵션 리스트 입력 (key : --executor-memory , value : 20G)
  17. forceOK : 실패시 강제 OK 처리여부
