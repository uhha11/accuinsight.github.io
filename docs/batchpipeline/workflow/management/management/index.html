워크플로우 관리

워크플로우 관리#

생성된 워크플로우의 목록을 조회하고, 바로 실행할 수 있다.

① 전체 워크플로우의 그룹, 상태, 워크플로우 명, 설명, 시간, 생성자 필드에 대한 키워드를 검색할 수 있다. 노드 명으로 해당 노드 타입을 가지는 워크플로우를 검색할 수 있으며, RDB, HIVE 노드의 경우 노드에 작성된 테이블명으로 워크플로우를 검색 할 수 있다.

② 버튼을 클릭하여 목록에서 체크한 워크플로우에 대해 전체 적용.

버튼명설명
전체 내보내기전체 워크플로우를 json 파일로 내보내기
내보내기선택한 n 개의 워크플로우를 json 파일로 내보내기
불러오기내보낸 워크플로우 json파일을 불러오기
실행선택한 n개의 워크플로우를 실행
정지선택한 n개의 워크플로우를 정지
삭제선택한 n개의 워크플로우를 삭제

③ 워크플로우 목록을 표시하며, 각 필드는 아래와 같은 의미를 가진다.

필드명설명
아이디워크플로우 ID
그룹워크플로우가 속한 그룹명
워크플로우 명워크플로우 이름
설명사용자가 작성한 Description
생성된 시각워크플로우를 생성한 시간
업데이트된 시각마지막으로 워크플로우를 수정한 시간
생성자워크플로우를 생성한 사용자
상태Active / Inactive
상세정보‘상세정보’ 화면이동 버튼, ‘인스턴스 목록’ 화면이동 버튼, ‘편집’ 화면이동 버튼
권한워크플로우의 권한 부여 팝업 버튼
Action 버튼실행, 종료, 삭제(*각 인스턴스 상태에 따라 그 시점에 사용 가능한 메뉴가 활성화됨)

④ 버튼을 클릭하여 해당 워크플로우의 상세정보를 확인하는 화면으로 이동

⑤ 버튼을 클릭하여 해당 워크플로우에서 발생한 인스턴스 목록을 확인하는 화면으로 이동

  • 인스턴스는 배치 인스턴스와 스케줄 인스턴스로 구분되며 각각 아이콘으로 구분된다.

⑥ 버튼을 클릭하여 해당 워크플로우의 편집화면으로 이동

⑦ 버튼을 클릭하여 해당 워크플로우의 권한을 편집하는 팝업 오픈

  • 워크플로우의 조회/기록/실행 권한을 다른 사용자에게 부여할 수 있다.

⑧ 버튼을 클릭하여 해당 워크플로우를 실행

  • 실행 시 잡 파라미터와 잡이 실행될 클러스터를 선택할 수 있다.

⑨ 버튼을 클릭하여 해당 워크플로우를 정지

⑩ 버튼을 클릭하여 해당 워크플로우를 삭제

⑪ 워크플로우를 실행한 이력이 있는 경우 가장 최근 실행한 인스턴스 상세 화면으로 이동
